Assevra GitHub PyPI Methodology Example

Measure the reliability of your LLM agents.

Assevra is an open-source toolkit and methodology that scores an agent on four reliability dimensions — each against a fixed threshold, with a confidence interval — and issues a signed reliability scorecard.

$ pip install assevra
View on GitHub See an example scorecard

Why

LLM agents fail quietly. They answer a question they should refuse, echo a record they should redact, or call a tool twice and no test catches it. A dashboard of averages hides these; a pass/fail gate against a labeled dataset does not. Assevra turns "the agent seems fine" into a number you can defend, with the sample size and interval attached so nobody over-reads a small run.

Four dimensions

llm-judge

Grounding

Does the answer follow from the provided context, or is it invented?

llm-judge

Safety / refusal

Does the agent refuse what it must refuse, and answer what it should?

deterministic

PII leak

Does sensitive data escape into an output? Zero tolerance on hard entities.

deterministic

Task completion

Did the agent actually do what was asked, end to end?

How it works

Point Assevra at a JSONL dataset of labeled cases. It routes each row to its dimension's scorer, aggregates the results into a scorecard with per-dimension pass rates and 95% Wilson intervals, and writes the report as Markdown, JSON, and a self-contained HTML file. The deterministic dimensions run offline; the judge dimensions use a pinned Claude model and are skipped, not failed, when no key is set. Add --gate and it exits non-zero on failure, so it gates a pull request directly instead of a dashboard nobody opens.
